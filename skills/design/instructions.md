# Design Skill

Use this skill for UI structure, interaction quality, accessibility, visual consistency, and product experience decisions.

Focus on:
- User goal, information hierarchy, and the primary workflow.
- Layout, spacing, alignment, responsive behavior, and content density.
- Accessibility: contrast, keyboard access, focus states, labels, alt text, and reduced motion.
- Interaction quality: loading states, error states, empty states, hover/press feedback, touch target size, and predictable navigation.
- Visual system: typography, color roles, icon usage, component consistency, and state styling.
- Implementation checks: avoid horizontal scroll, layout shift, clipped text, unlabeled controls, emoji-as-icons in product UI, and viewport-scaled typography.

For design review, return:
1. Highest-impact issues first.
2. Concrete fixes, not broad taste notes.
3. Responsive and accessibility risks.
4. What should be verified in browser or screenshots.

For design implementation, prefer familiar controls and stable dimensions. Use existing project components and design tokens when present.
